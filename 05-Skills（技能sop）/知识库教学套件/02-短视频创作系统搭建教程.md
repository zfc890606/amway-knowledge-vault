# 🎬 02 · 短视频创作系统搭建教程

> **为什么你需要一套短视频创作系统？**
>
> 很多人做短视频是这样的：今天想到一个选题就拍一个，下周没灵感了就断更。这不是创作，是碰运气。
>
> 真正的创作者有**系统**——选题有来源、脚本有模板、质量有检查、发布有节奏。你不用靠灵感吃饭，你靠系统生产。
>
> **前提**：建议先完成 [[01-知识库搭建教程]]，因为短视频的素材来自你的知识卡片。

## 核心概念：三阶段管线

短视频创作不是"写稿 → 拍 → 发"这么简单。完整的管线分三个阶段：

```
┌─ Phase 1：调研 ──────────────────────┐
│  确定做什么选题 → 验证热度 → 看同行    │
│  产出：明确选题 + 参考素材              │
└───────────────────────────────────────┘
                    ↓
┌─ Phase 2：脚本生产 ───────────────────┐
│  知识卡片拼装 → 写稿 → 风格化 → 质量检查│
│  产出：一条可以直接拍的脚本              │
└───────────────────────────────────────┘
                    ↓
┌─ Phase 3：发布与复盘 ─────────────────┐
│  拍摄 → 发布 → 追踪数据 → 迭代优化     │
│  产出：数据反馈 + 下一条改进方向         │
└───────────────────────────────────────┘
```

## 第一阶段：调研与选题

### 1.1 你的选题从哪里来

好的选题不是"我觉得这个会火"，而是从这些地方来的：

| 来源 | 怎么找 | 例子 |
|------|--------|------|
| **客户评论区** | 去你的同行/对标账号评论区"偷听" | "长客会怎么续约"这个痛点是评论区发现的 |
| **客户问你的问题** | 同一个问题被问 3 次以上 → 就是好选题 | "脂肪肝能吃蛋白粉吗" |
| **搜索关键词** | 在搜一搜 / 百度 / 小红书搜你的领域 | "安利新手第一步做什么" |
| **热点话题** | 借势但不蹭热度 | "这个全民话题跟营养有什么关系" |

### 1.2 验证：这个选题值得做吗？

找到 5 个以上潜在选题后，用这个清单过滤：

```
□ 这个问题痛不痛？（被反复问？不解决会怎样？）
□ 看的人多不多？（搜索量怎么样？）
□ 我能给出独特答案吗？（不是复制别人的）
□ 我有没有相关素材/知识卡片？（有最好，没有要补）
□ 拍出来会不会违规？（视频号/抖音的审核红线）
```

> 一个选题过 5 关就是好选题。不过的暂时放一边。

### 1.3 建立你的痛点排行榜

建一个表格（用 Obsidian 表格或 Excel），记录所有发现的问题：

| 排名 | 痛点 | 来源 | 频率 | 类型 |
|:----:|------|:----:|:----:|:----:|
| 1 | 安利做不起来/总是失败 | 评论区 | ⭐⭐⭐⭐⭐ | 痛点共鸣型 |
| 2 | 新手第一步做什么 | 搜索+客户问 | ⭐⭐⭐⭐⭐ | 方法教学型 |
| 3 | 长客会怎么推广/续约难 | 评论区 | ⭐⭐⭐⭐ | 方法教学型 |
| ... | ... | ... | ... | ... |

> 排行榜就是你的**拍摄计划**——从第一名开始拍，拍完一个往下挪一个。

## 第二阶段：脚本生产流水线

### 2.1 知识卡片拼装法

每条视频脚本 = 几张知识卡片拼在一起。你不需要每次都从零开始写。

**举个例子**：

```
你要写"安利新手第一步做什么"的脚本
→ 找到相关卡片：
   • KC-创业-安利新人常见误区
   • KC-营养-蛋白质每日摄入量
   • KC-创业-高频痛点筛选法
→ 拼成一条脚本：痛点代入 → 认知翻转 → 给方法 → 验证结果
```

### 2.2 六种视频结构模板

有 6 种经过验证的脚本结构，你直接套用，不用自己发明：

| 类型 | 情绪曲线 | 适合场景 |
|:----:|---------|---------|
| **A 痛点共鸣型** | 扎心 → 共情 → 翻转 → 再扎 → 给解法 → 收尾 | "我也经历过，我懂你" |
| **B 方法教学型** | 质疑 → 踩坑 → 发现问题 → 给方法 → 收尾 | "三步搞定XXX" |
| **C 认知颠覆型** | 抛出问题 → 打破 → 解释 → 对号入座 → 结论 | "90%的人都搞反了" |
| **D 营养科普型** | 焦虑 → 病因诊断 → 认知反差 → 给方案 → 温暖收尾 | "脂肪肝不是吃太多油" |
| **E 案例见证型** | 背景 → 转折 → 结果 → 方法 → 邀请 | "我三个月变化这么大" |
| **F 答疑解惑型** | 提问 → 直接回答 → 展开解释 → 建议收尾 | "你问的这个问题..." |

> 选结构的方法：看你的选题是哪种类型，直接对应进去。

### 2.3 开头 3 秒钩子

开头决定完播率。不用铺垫，直接上钩子：

**九大人性钩子（选一个用）**：

| 钩子类型 | 例子 |
|---------|------|
| 👁️ **视觉** | "看这张体检单" |
| 💰 **赚钱** | "我月入 2500 到年入 150 万，只改了一件事" |
| 😨 **恐惧** | "你这三种习惯，正在毁掉你的肝脏" |
| 😤 **愤怒** | "我特别烦那种一上来就让你买买买的" |
| 🗣️ **八卦** | "我认识的年入百万的安利人，没有一个靠卖货" |
| 🔍 **猎奇** | "安利不靠卖货赚钱，你信吗" |
| 💎 **高价值** | "这条视频值你一年的营养知识学费" |
| 💵 **炫富** | "我做安利第一个月赚了 80 块，但那是转折点" |
| 🔥 **热点** | "最近这个很火的减肥法，劝你别试" |

### 2.4 脚本写作公式

每条脚本的基本骨架：

```
开头（3-5秒）：钩子 → 制造"嗯？"
  ↓
中段（20-40秒）：痛点 → 方案 → 原因
  ↓
结尾（5-10秒）：一句话总结 + CTA（行动号召）
```

## 第三阶段：发布与复盘

### 3.1 发布前检查清单

```
□ 脚本风格自检（见下文）
□ AI 痕迹检测（读起来像不像人话？）
□ 平台审核风险排查（有没有敏感词？）
□ 标题 + 封面准备好了吗？
□ CTA 清晰吗？（关注/评论/私信？）
```

### 3.2 发布后数据追踪

建立数据追踪表，每发一条记录一条：

| 视频 | 发布时间 | 播放量 | 点赞 | 评论 | 转发 | 粉丝增长 |
|:----:|:--------:|:-----:|:----:|:----:|:----:|:-------:|
| #001 安利做不起来 | 07-15 | — | — | — | — | — |

> 够 10 条数据后再做分析。一两周的数据看不出规律。

### 3.3 复盘迭代节奏

| 频率 | 做什么 | 看什么指标 |
|:----:|--------|----------|
| 发完 3 天后 | 填数据追踪表 | 所有数据 |
| 每 10 条 | 做一次内容复盘 | 哪条播放最高/最低？像什么？ |
| 每月 | 调整策略 | 痛点排行榜要不要更新？ |

## 脚本风格系统：怎么写出"人话"

写脚本最怕的是"AI 味"——听起来像说明书、像教科书、像 AI 写的。以下几条帮你写出"人话"。

### 12 条铁律

1. **开头不要铺垫，直接扎心** — 第一句话就是钩子
2. **第一句话必须是"嘴替"** — 替观众说出他心里的那句
3. **用具体场景代替抽象概念** — 不是"营养很重要"而是"你每天缺了 20 克蛋白"
4. **用"否定 → 打破 → 重建"替代平铺直叙** — 先推翻他的认知，再给新的
5. **每 30 秒制造一个"嗯？"** — 让他觉得"这我怎么没想到"
6. **三个观点递进，不并列** — 1+1+1 > 3，不是罗列三点
7. **多用对话，少讲道理** — "你问他：你是不是也这样？"
8. **敢说狠话** — "不是不适合，是懒"
9. **结尾笃定，不煽情** — 不是"希望能帮到你"而是"照做就行了"
10. **每条 CTA 不一样** — 形成连续剧，让人觉得下一条更精彩
11. **不用"所以"开头** — 口语里没人这么说
12. **写完读一遍—像不像人话？** — 读出声来是检验标准

### 风格自检 5 关

写完后，逐条问自己：

```
□ 创作本质关：这条视频有没有给观众新的认知？还是说了一堆废话？
□ 嘴替关：观众看完会说"对，我就是这么想的"吗？
□ 自信关：说话的语气够笃定吗？有没有"可能/也许/大概"？
□ 情绪关：情绪曲线对吗？有没有让观众从→到→的体验？
□ 真实感关：读出声来自然吗？还是像在念稿？
```

## 质量检测：发之前必过 3 关

### 第 1 关：去 AI 味

写完后，大声读一遍。读到不顺的地方就改。AI 写的句子通常：

- 太长（超过 15 个字没逗号 → 拆成短句）
- 太完美（正常人说话有停顿、重复、语气词）
- 太多连接词（"因此""然而""此外" → 去掉）

### 第 2 关：内容风险检测

过一遍视频号/抖音的审核红线：

```
□ 医疗领域：没有说"治病""疗效"，说的是"调理""改善"
□ 产品推广：没有过度承诺，没有对比贬低
□ 金钱话题：没有炫富，没有保证收益
□ 敏感词：没有违规词汇
```

> 当你用了 AI 工具时，可以用 content-risk-detector 技能自动检测。

### 第 3 关：真人复核

找一个人（你的伙伴或家人），让他看完回答三个问题：

1. 你看完记住了什么？（检验核心信息是否清晰）
2. 你有想评论的冲动吗？（检验有没有互动点）
3. 你相信吗？（检验真实感）

## 常见问题

### Q: 我没有知识卡片，能直接写脚本吗？
可以。先按第二步的六种结构写脚本。拍 10 条后再回头建知识库也来得及。但长期看，有卡片库会让写脚本快 3 倍。

### Q: 写一条脚本要多久？
系统熟练后：选题 5 分钟 → 写稿 20 分钟 → 检查 5 分钟 = 30 分钟一条。第一条可能会花 1–2 小时，正常。

### Q: 开头钩子写不出来怎么办？
用"你知道吗"开头保险，"我以前也..."回忆式开头也有效。或者最直接：**提一个反常识观点**——"我跟你讲个事，做安利不是先卖货"。

### Q: 拍了没人看怎么办？
正常。没人看只有两个原因：要么选题不够痛，要么开头不够钩。连拍 10 条，对比数据，你会找到感觉的。如果 10 条后数据还不行，回到第一阶段重新调研选题。

### Q: 我该用 AI 帮我写脚本还是自己写？
都行。但 AI 写完后你一定要**改**——AI 理解不了你的真实经历和语气。参考流程：AI 出骨架 → 你填入真实经历 → AI 帮忙优化表达 → 你读出声改到顺为止。
