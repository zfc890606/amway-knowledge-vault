# 🌐 04 · 网络配置教程（梯子）

> **为什么你需要这个？**
>
> 我们的工具链中，Claude Code（Anthropic API）和部分技能的安装源（GitHub）需要稳定的国际网络连接。没有这个，安装可能会卡住。
>
> 这不是什么"翻墙教程"——这只是为你配置必要的网络环境，让 AI 工具能正常为你工作。
>
> **适用场景**：
> - 安装 Node.js 时下载不动
> - 注册 Anthropic API 时打不开网页
> - 用 `npx skills add` 安装技能时超时
> - 使用 Claude Code 时响应慢或断连

## 你需要什么样的网络工具

| 场景 | 需要 | 不需要 |
|------|------|--------|
| 打开 GitHub / Anthropic 网页 | 浏览器代理 | — |
| 安装 Node.js / npm 包 | 终端代理 | — |
| Claude Code 正常工作 | 终端代理 | — |
| DeepSeek（国内可用） | ❌ 不需要 | 国内直连 |
| Obsidian 同步 | ❌ 不需要 | 国内直连 |
| 刷抖音/小红书/视频号 | ❌ 不需要 | 国内直连 |

> **一句话**：只有 Claude Code、GitHub、Anthropic 这三个东西需要代理，其他全部国内直连。

## 第一步：选择并安装 VPN/代理工具

### 选项 A：Clash 系列（推荐，配置灵活）

Clash 是目前最常用的代理工具，支持 Windows / Mac / 手机。

**安装步骤**：

1. 打开浏览器，访问 Clash 客户端下载页面（让推荐给你的人发链接，或搜索"Clash 客户端下载"）
2. 下载对应系统的版本：
   - Windows 用户：下载 `Clash.for.Windows.zip` 或 `Clash-verge`
   - Mac 用户（Intel 芯片）：下载 `ClashX.dmg`
   - Mac 用户（M1/M2/M3 芯片）：下载 `ClashX.dmg` 或 `Clash-verge`（arm64 版）
3. 下载后安装（Mac 用户拖入 Applications 文件夹，Windows 用户解压后运行 .exe 文件）
4. 导入配置文件（你的服务商会给你一个"订阅链接"或"配置文件.yaml"，在客户端中导入）
5. 开启代理（开启"系统代理"开关）

**验证**：在浏览器中访问 `https://www.google.com` 或 `https://github.com`，如果能打开，说明代理正常工作。

### 选项 B：VPN 类工具（更简单但灵活性低）

如果 Clash 太复杂，也可以使用 VPN 类工具。

1. 选择你信得过的 VPN 服务
2. 下载 VPN 客户端
3. 注册并登录
4. 选择可访问国际网络的节点并连接

> ⚠️ **选择标准**：
> - 选择信誉好、稳定的服务
> - 不要用免费的——免费的往往不稳定或不安全
> - 问用过的伙伴推荐

## 第二步：配置网络代理

不是装好 VPN/代理工具就完事了。你还需要告诉电脑上的**终端**（Terminal / CMD）和**其他工具**"走代理出去"。

### 2.1 开启代理工具的"允许局域网连接"

大多数代理客户端有一个"Allow LAN"或"允许局域网"的选项——**打开它**。这样终端和工具才能使用代理。

### 2.2 查看代理端口

打开你的代理客户端，找到"HTTP 代理端口"或"混合代理端口"。常用的端口是：

| 代理工具 | 默认 HTTP 端口 | 默认 SOCKS5 端口 |
|---------|:-------------:|:----------------:|
| ClashX | 7890 | 7891 |
| Clash-verge | 7897 | 7898 |
| V2RayU | 1087 | 1086 |

> 记下你的 HTTP 端口号（通常是 7890 或 7897），后面需要用到。

## 第三步：在各工具中配置代理

### 3.1 终端代理配置（让命令行能访问国际网络）

#### Mac / Linux 用户

在终端中执行以下命令（把 `7890` 换成你的实际端口）：

```bash
# 一键配置代理环境变量（临时生效，关闭终端后失效）
export https_proxy=http://127.0.0.1:7890
export http_proxy=http://127.0.0.1:7890
export all_proxy=socks5://127.0.0.1:7891
```

> 每次新开终端都需要重新执行。如果你希望永久生效，把上面三行加到 `~/.zshrc` 文件中。

#### Windows 用户

在命令提示符或 PowerShell 中执行：

```bash
# 命令提示符
set https_proxy=http://127.0.0.1:7890
set http_proxy=http://127.0.0.1:7890

# PowerShell
$env:https_proxy="http://127.0.0.1:7890"
$env:http_proxy="http://127.0.0.1:7890"
```

### 3.2 npm 代理配置（安装 AI 技能时需要）

如果你的 npm（Node.js 的包管理器）也需要代理：

```bash
# 配置 npm 使用代理
npm config set proxy http://127.0.0.1:7890
npm config set https-proxy http://127.0.0.1:7890

# 验证配置
npm config get proxy
npm config get https-proxy
```

### 3.3 Git 代理配置（从 GitHub 克隆技能时需要）

```bash
# 配置 Git 使用代理
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890

# 验证配置
git config --global --get http.proxy
git config --global --get https.proxy
```

### 3.4 Claude Code 网络配置

Claude Code 的 API 调用会走系统代理。如果你已经按上面的步骤配置了终端代理，它一般会自动生效。

如果还是连不上，在终端启动 Claude Code 前先执行：

```bash
export ANTHROPIC_BASE_URL=https://api.anthropic.com
```

或者你在 Claudian 插件设置中，看看有没有"代理"相关的配置项，填入你的代理地址。

## 第四步：验证网络配置正确

### 4.1 浏览器验证

打开浏览器，测试以下网址是否能正常访问：

- [ ] `https://github.com` — 代码托管平台（必测）
- [ ] `https://console.anthropic.com` — AI API 控制台（必测）
- [ ] `https://www.google.com` — 搜索引擎

> 打不开 → 检查代理客户端是否开启、端口是否正确、是否开启了系统代理开关。

### 4.2 终端验证

在终端中执行以下命令：

```bash
# 方法一：用 curl 测试
curl -I https://www.google.com

# 方法二：用 ping 测试
ping -c 3 github.com
```

如果返回正常响应（HTTP 200 或 ping 有返回），说明终端代理配置成功。

### 4.3 npm 和 Git 验证

```bash
# 测试 npm
npm ping

# 测试 Git
git ls-remote https://github.com/anthropics/claude-code.git HEAD
```

### 4.4 全流程验证

做完以上所有步骤后，尝试一条完整的安装命令测试：

```bash
npx skills add https://github.com/op7418/Humanizer-zh.git
```

如果正常运行并显示安装成功，恭喜——你的网络配置全部正确！

## 故障排除

### 问题 1：浏览器能打开 Google，但终端连不上

终端不会自动走代理，需要：**先执行第三步的终端代理配置命令**。每次新开终端都要执行一次。

### 问题 2：curl 测试报错 "Connection refused"

```
原因：终端代理端口配置不对
解决：
  1. 确认代理客户端正在运行（检查菜单栏/任务栏图标）
  2. 确认端口号正确（在代理客户端设置界面查看 HTTP 端口）
  3. 确认代理客户端中"允许局域网连接"已开启
```

### 问题 3：npm 安装速度极慢

```bash
# 尝试换用国内镜像（但注意：安装 GitHub 上的技能时还是要走代理）
npm config set registry https://registry.npmmirror.com

# 需要走代理时再切回来
npm config set registry https://registry.npmjs.org
```

### 问题 4：Claude Code 回复很慢或超时

```
可能原因：
  1. 代理连接不稳定 → 换一个节点试试
  2. API Key 额度用完了 → 登录 Anthropic 控制台检查
  3. 网络波动 → 等 5 分钟后重试
```

### 问题 5：手机端也需要用？

安装手机版代理客户端（Clash 有 iOS/Android 版），用同样的订阅链接配置即可。手机端主要用于 Claude Code App 或其他 App。

### 问题 6：代理突然不能用了

```
第一步：检查你的代理服务是否到期
第二步：检查代理客户端是否更新（有些旧版本会失效）
第三步：联系你的服务商看是不是线路问题
第四步：找伙伴推荐一个替代方案
```
