# 微信视频号内容获取方法

> **一句话本质**：视频号是封闭生态，无法直接通过常规浏览器访问，需借助 在线API / 本地代理 / 轻量工具 三类方案间接获取。

---

## 🔍 为什么没法直接访问

- **封闭生态**：视频号是微信生态内的平台，内容需要微信登录态，没有公开网页接口
- **加密保护**：视频前 131072 字节经过 XOR 加密，需要 `decode_key` 解密才能播放
- **CDN 链接过期**：视频直链和 `decode_key` 成对返回，每次请求不同，必须配对使用

---

## 方案总览

| 路线 | 适合谁 | 代表工具 |
|------|--------|----------|
| **在线 API 服务** | 开发者、批量/自动化需求 | Apify、TikHub |

| **轻量公众号/网页工具** | 普通用户，偶尔下载 | 犀牛下载器、DOWNNI |
| **低代码自动化** | n8n 用户 | n8n-nodes-video-crawler |

## 方案详解

### 一、在线 API 服务

#### 1. Apify — WeChat Channels Scraper

- 无需微信账号、无需 API Key
- 支持：关键词搜索、按创作者拉视频列表、exportId 解析为直链
- 付费按结果计费，提供 Python SDK
- 链接：https://apify.com/sian.agency/wechat-channels-scraper

#### 2. TikHub.io — WeChat Channels V2 API

接口更全，覆盖：

| 接口 | 用途 |
|------|------|
| `fetch_video_detail` | 视频详情 + 解密密钥 |
| `fetch_user_videos` | 按创作者列出视频 |
| `fetch_user_profile` | 粉丝数、简介、关联账号 |
| `fetch_live_history` | 直播回放 + 下载密钥 |
| `fetch_search_channel_videos` | 号内搜索 |

- 鉴权：Bearer Token
- 视频下载：`url + url_token` 拼接成完整 CDN 链接，用 `decode_key` XOR 解密前 131072 字节
- 注意：返回的 ID 是 64 位整数，需用字符串处理（JSON-BigInt）避免精度丢失
- 链接：https://docs.tikhub.io/

---

### 二、自部署开源工具

#### 1. scribe-transcribe（Python + Go）

- **原理**：MITM 代理拦截，自动在视频号网页注入下载按钮
- **操作**：浏览器打开 `channels.weixin.qq.com` 即可点击下载
- **支持**：Windows / macOS / Linux
- **可选**：在手机微信 App 内操作（需配置代理）
- **前置依赖**：Go 1.21+、Python 3.10+
- **GitHub**：https://github.com/jun7799/scribe-transcribe

#### 2. wx_channel（Go）

- **原理**：本地 HTTP 代理 + 脚本注入
- **特点**：零配置，下载即用，默认端口 `2025`
- **功能**：批量下载、智能去重、加密视频解密、多格式导出（TXT / JSON / Markdown）
- **GitHub**：https://github.com/yuanqiying/wx_channel-20251110copy

## 通用技术原理

```
搜索/列表 → 返回 exportId / object_id
        ↓
获取视频详情 → 返回 media.url + media.url_token + media.decode_key
        ↓
拼接 CDN 链接 → url + url_token
        ↓
XOR 解密 → decode_key 解密前 131072 字节 → 可播放 MP4
```

⚠️ `decode_key` 和签名 URL 每次请求都会刷新，即使是同一个视频——必须从同一次响应中配对使用。

## 场景推荐

- **一次下载一两个视频** → 犀牛下载器 / DOWNNI（最简单）
- **批量抓取某个创作者的全部视频** → Apify / TikHub API
- **搭建自动化工作流** → n8n-nodes-video-crawler
- **技术向自部署，完全可控** → scribe-transcribe / wx_channel
- **任何方法都失效** → 屏幕录制兜底
