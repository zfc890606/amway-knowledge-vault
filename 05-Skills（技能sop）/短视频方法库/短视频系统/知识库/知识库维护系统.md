## 二、维护流程（三步走）

### 第1步：📋 扫描 → 出报告

自动执行以下扫描，生成一份 **维护报告**：

| 扫描项 | 检测内容 | 报告示例 |
|--------|---------|---------|
| **重复文件检测** | 文件名相似、内容高度重复的脚本 | `⚠️ script-002 有两个版本，建议保留一个` |
| **空文件夹检测** | 没有任何文件的空目录 | `⚠️ 日记/ 目录为空` |
| **临时文件检测** | `.DS_Store`、`Thumbs.db` 等 | `ℹ️ 发现 5 个 .DS_Store，共 30KB` |
| **大文件检测** | 超过 5MB 的非媒体文件 | `ℹ️ 86%的糖尿病可逆.pptx → 17MB，建议归档` |
| **过时检测** | 超过60天未更新的非核心文件 | `⚠️ 文件 X 最后更新于 XX，是否归档？` |
| **已拍摄脚本** | 脚本 frontmatter 中 status=已拍摄 的条目 | `ℹ️ 已有 3 条脚本标记为已拍摄` |

### 第2步：✅ 你确认 → 标记

> ⚠️ **绝不自动删除任何文件。**
>
> 报告出来后，每项操作都需要你确认：
> - `确认归档` → 移到 `04-Archive/`
> - `确认保留` → 打标记，跳过
> - `确认删除` → 移入 trash（先不永久删除）

### 第3步：🚀 执行 → 记录

执行完成后，在 `知识库/日志.md` 中追加一条执行记录：

```
---
维护日期：2026-07-14
执行动作：
  - 归档：2 个脚本（已拍摄）
  - 清理：3 个 .DS_Store
  - 保留：1 个重复文件（用户确认保留）
---
```

## 三、各区域清理规则

### 📁 scripts/ 脚本区

| 规则 | 说明 |
|------|------|
| **同名去重** | 完整版 + 纯文案版同时保留，命名标准化后无重复 |
| **已拍摄归档** | frontmatter `status: 已拍摄` 的脚本，移动到 `04-Archive/短视频脚本/` |
| **纯文案 vs 完整版** | 同时保留（纯文案对口播，完整版带备注供参考），不做合并 |
| **命名规范** | 统一为 `script-NNN-标题.md` 格式，不要含 `？` 和标点 |

### 📚 03-Resources/ 参考素材区

| 规则 | 说明 |
|------|------|
| **4板块归类** | 顶层按01-营养/02-创业/03-短视频/04-带团队归类，子目录保持原位 |
| **文件格式标准化** | .docx 文件转成 .md（用 markitdown skill），转后原件可归档 |
| **重复素材合并** | 同一人物/话题的多个文件，合并为一个完整档案 |
| **板块目录索引** | 每个板块目录下的 index.md 是否包含新增内容 |
| **过时话题归档** | 如果某个话题不再关注，移到 Archive |

### 🎯 00-灵感库/ 灵感区

| 规则 | 说明 |
|------|------|
| **已用灵感标记** | 已经写成脚本的灵感卡片，在 frontmatter 加 `status: used` |
| **3个月未用的灵感** | 移到 Archive，不删 |
| **知识卡片** | 持续积累，不清理 |

### 📤 06-输出/ 成品区

| 规则 | 说明 |
|------|------|
| **大文件归档** | 超过 10MB 的 PPTX 文件，已使用的移到 Archive |
| **临时构建文件** | 用于测试/实验的输出，用完后标记清理 |

### 🗑️ 全 vault 通用

| 规则 | 说明 |
|------|------|
| **空文件夹** | 持续空置超过30天 → 询问是否清理 |
| **.DS_Store** | 每次维护清理一次，不询问 |
| **孤立文件** | 没有被任何笔记引用的孤立笔记 → 报告但不删除 |
| **超过90天未更新** | 非核心文件 → 报告是否归档 |

## 五、30天自动触发设置

> 本系统由 CronCreate 每30天自动触发。
>
> **触发时自动执行：**
> 1. 读本文件 → 了解维护规则
> 2. 执行第1步扫描 → 生成维护报告
> 3. 输出报告 + 询问你确认
> 4. 按你的确认执行操作
> 5. 更新日志.md

### 因为CronCreate任务7天后自动过期，实际执行方式：

| 方式 | 说明 |
|:----|------|
| **首次** | 我现在就出一份扫描报告，把已知问题先清理掉 |
| **30天后** | 自动触发提醒，我重新出报告 |
| **持续** | 每次触发时，如果任务已过期，你跟我说"维护"或"整理"，我重新设 |
