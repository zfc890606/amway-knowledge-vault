## 三条路径对比

| 对比项 | 🅰️ wewrite 全自动管线（推荐） | 🅱️ 手动粘贴（旧） | 🅲 API 自动上传（进阶） |
|-------|---------------------------|-------------------|---------------------|
| **配置成本** | 安装 wewrite + 配风格即可 | 零配置 | 需公众号开发者权限 |
| **操作步骤** | 一句话触发 → 全自动完成 | 转换 → 复制 → 粘贴 | 配置后一键推送 |
| **排版主题** | ✅ 18 个主题（bold-navy 等） | ❌ 无样式 | ⚠️ 部分保留内联样式 |
| **风格一致性** | ✅ style.yaml 统一控制 | ❌ 手动调整 | ✅ style.yaml 统一控制 |
| **推荐度** | ⭐⭐⭐⭐⭐ **推荐** | ⭐⭐⭐ 备选 | ⭐⭐⭐ 有API时可用 |

## 🅰️ 推荐：wewrite 全自动管线

### 前置安装（一次性）

```bash
# 已安装则跳过
bash /Users/mac/Documents/.claude/repos/wewrite/install.sh
```

### 风格配置

wewrite 使用 `~/.wewrite/style.yaml` 控制公众号风格，包括：

- **写作人格**：industry-observer / midnight-friend / warm-editor 等 7 种
- **排版主题**：bold-navy / professional-clean / warm-editorial 等 18 种
- **语气风格**：专业分析 / 犀利观点 / 温暖叙事 等

> 完整主题列表 → `wewrite gallery` 可浏览器预览

当前配置（[[操作系统总控台]] 的定位）：
```yaml
name: "张大侠在进化"
theme: bold-navy          # 藏青稳重，适合行业分析
writing_persona: industry-observer  # 专业分析 + 犀利观点
```

### 创作流程（一句话触发）

**完整管线（从痛点 → 文章 → 预览）：**

```
你发指令
   │
   ▼
┌─ wewrite diagnose ──────────────────────────────────┐
│  检查依赖、风格文件、API 配置                       │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌─ wewrite run start ─────────────────────────────────┐
│  建任务：选模式 ── mode publish（推草稿箱）          │
│                ── mode complete（本地预览+排版）      │
│                ── mode draft（仅初稿）               │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌─ wewrite-write ─────────────────────────────────────┐
│  读选题/痛点 → 任务书(brief) → 主张与证据(claims)   │
│  → 初稿(draft) ← 调用写作人格 + 洞见卡金句          │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌─ wewrite-review ────────────────────────────────────┐
│  事实核查 × 观点审校 × 实用性 × 账号声音 × 可读性    │
│  → 通过后封存为 article.md                          │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌─ wewrite preview ───────────────────────────────────┐
│  Markdown → 主题排版(bold-navy等) → HTML 本地预览    │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌─ 交付 ──────────────────────────────────────────────┐
│  ├─ 文章存 → 00-灵感库/公众号草稿/                  │
│  ├─ 预览存 → 同目录 -preview.html                   │
│  └─ 有API密钥 → 可选推公众号草稿箱                   │
└──────────────────────────────────────────────────────┘
```

**触发方式（你说 → 我自动执行）：**

| 你说 | 动作 |
|------|------|
| **"写一篇公众号文章，选题是XX"** | 读痛点排行榜+知识卡片 → 全管线 → 存 公众号草稿/ |
| **"把脚本X改成公众号文章"** | 读脚本+知识卡片 → 口播→阅读体改写 → 质检 → 存 公众号草稿/ |
| **"把痛点#XX写成公众号"** | 读痛点详情+关联知识 → 按痛点评分适配结构 → 全管线 |
| **"推到草稿箱"** | 已有完成文章 → 配置了API则直接推送，否则生成本地预览 |
| **"换主题/换风格"** | `/wewrite-style` 重新配置 style.yaml |

### 质检管线

每次公众号文章自动过检（写入文章前）：
```
humanizer-zh → content-risk-detector → ai-detector
```

## 🅱️ 旧路径A：Markdown → WeChat 手动粘贴（备用）

当 wewrite 不可用时，使用此路径。

### 工具

`.claude/skills/wechat-publisher/convert.py` — Python 脚本，将 Markdown 转为公众号可粘贴的 HTML。

### 操作步骤

```
1. 按 [[公众号文章模板]] 完成文章
   └─ 存到 00-灵感库（标记灵感）/公众号草稿/

2. 转换格式
   └─ python3 .claude/skills/wechat-publisher/convert.py "00-灵感库（标记灵感）/公众号草稿/article-xxx.md"
   └─ 脚本自动将 HTML 复制到剪贴板（macOS）

3. 粘贴到公众号后台 → 保存草稿
   └─ 打开 https://mp.weixin.qq.com → 新建图文
   └─ Cmd+V 粘贴 → 检查格式 → 补充封面图/摘要
   └─ 保存为草稿

4. 自行发布（等图片准备好后）
```

### 转换对照

| Markdown | → | WeChat 效果 |
|----------|---|-------------|
| `# 标题` | → | H1 大标题 |
| `## 分段` | → | H2 分段标题 |
| `**加粗**` | → | **加粗** |
| `> 引用` | → | 引用块 |
| `-`/`1.` 列表 | → | 列表 |
| `---` | → | 分隔线 |

## 🅲 旧路径B：API 自动上传草稿箱（进阶）

> 需在公众号后台配置开发者权限。配置后 wewrite 可直接推草稿箱。

### 配置步骤

```
公众号后台 → 设置与开发 → 基本配置
├─ AppID:  直接复制
├─ AppSecret: 重置后记下来
└─ IP 白名单: 添加本机公网 IP
```

拿到以下值后告诉我：
```
AppID:        wx...
AppSecret:    xxxxxxxxxx...
thumb_media_id:  封面图的 media_id
```

### 配置后效果

你说"写一篇公众号文章，推到草稿箱"
→ wewrite 全自动：写作 → 审稿 → bold-navy 排版 → 推入公众号草稿箱
→ 你去 mp.weixin.qq.com → 草稿箱 → 加封面图 → 发布

## 文章存储规范

### 目录结构

统一存放在 `00-灵感库（标记灵感）/公众号草稿/`：

```
00-灵感库（标记灵感）/公众号草稿/
├── 2026-07-17-视频号还来得及吗.md     ← wewrite 产出
├── 2026-07-17-视频号还来得及吗-preview.html  ← 预览文件
├── article-005-标题.md                ← 旧路径产出
└── ...
```

### 命名规范

```
wewrite 产出：YYYY-MM-DD-标题.md（日期前缀）
旧路径产出：article-XXX-标题.md（编号前缀）
```

### Frontmatter 约定

```yaml
---
title: 文章标题
created: 2026-07-17
updated: 2026-07-17
tags: [wechat-article, 板块]
status: draft          # draft | published
wechat_url:            # 发布后补充
cover_image:           # 封面图路径
source_script:         # 如果由脚本转化，填写脚本编号
generator: wewrite     # wewrite | manual
theme: bold-navy       # 排版主题（wewrite 产出时）
---
```

### 状态流转

```
写作中 → 草稿（存公众号草稿/） → 已发布
  │            │
  │      在公众号后台保存草稿后更新
  │
 wewrite 产出时自动存储
```

## 常见问题

### Q：wewrite 推草稿箱后手机上没看到主题样式？

微信公众号手机端会砍掉部分 CSS（如 border-left、padding）。wevwrite 已做了最大兼容处理（内联样式 + data-darkmode 属性），但完全一致的效果需要以下任一路径：
1. **配 API 推草稿箱**（内联样式保留度最高）
2. **本地预览后截图参考**，手动在公众号编辑器排版

### Q：图片怎么处理？

目前 wewrite 不自动配图。文章完成后说"配个封面"或"配图"进入视觉模块。
手动路径：上传到公众号素材库后插入。

### Q：旧脚本 convert.py 还用吗？

作为 wewrite 不可用时的备选。wewrite 安装后自动接管公众号创作管线。
