# 📱 公众号文章模板

> 类比 [[知识库/05-Skills（技能sop）/短视频方法库/短视频系统/知识库/脚本模板|脚本模板]]，专门为 WeChat 公众号排版设计。
> 写之前读 [[知识库/05-Skills（技能sop）/短视频方法库/短视频系统/知识库/脚本风格系统|脚本风格系统]] 底层的嘴替逻辑，但表达方式从口播体切换为阅读体。

---

## 短视频脚本 → 公众号文章 转化要点

| 短视频脚本 | 公众号文章 |
|-----------|-----------|
| 无标题 | H1 `# 标题` — 吸引点击，15-20字 |
| 靠情绪曲线推进 | H2/H3 分段 `##`/`###` — 逻辑清晰可扫读 |
| 口语重音 | `> 引用` 突出金句 — 读者视线落脚点 |
| 语气强调 | `**加粗**` 强调核心观点 |
| "第一第二第三"口述 | `-`/`1.` 列表 — 结构化呈现 |
| 短句（口语体） | 较长段落但不过密（每段3-5行为宜） |
| 结尾一句话 | 结尾 CTA 段落 — 引导关注/评论/转发 |
| 纯口播 | 可配封面图+导语（文章摘要） |

---

## 通用结构模板

### 标题（H1）

> 15-20 字，含关键词 + 好奇心/利益点
> 例：*做安利第一步就搞反了？90%的新手都栽在这*

### 导读（可选，斜体）

> 2-3 句引出痛点或问题，让读者产生"说的就是我"的代入感
> *你是不是也觉得做安利就是不停卖产品？我告诉你，真正做起来的人，根本不是这么干的。*

---

### 正文分段

```
## 第一段：痛点/场景切入（H2）
  延续标题的钩子，3-5句把读者的处境说透。
  用「你」开头，保持对话感。

## 第二段：核心方法/观点（H2）
  展开 2-3 个要点，每点用 H3 或列表：
  
  ### 要点一：XXXX
  展开说明，可以用 **加粗** 强调关键词。
  > 金句引用 — 读者扫读时视线落在这

  ### 要点二：XXXX
  同上结构。

## 第三段：底层逻辑/原因（H2）
  为什么这个方法有效？给他一个认知框架。
  不用太长，3-4句讲透即可。

## 结尾：总结 + CTA（H2）
  一句话总结全文核心。
  
  > 金句收束
  
  💬 你卡在哪一步？评论区告诉我。
  📌 觉得有用点个「在看」，转发给需要的朋友。
```

## 排版规范

| 元素 | 使用场景 | 格式 |
|------|---------|------|
| `#` H1 | 标题（全文唯一） | `# 标题文字` |
| `##` H2 | 主要段落开头 | `## 段落标题` |
| `###` H3 | 子要点 | `### 子标题` |
| `> 引用` | 金句/重点结论 | `> 金句内容` |
| `**加粗**` | 关键术语/核心观点 | `**关键词**` |
| `-` 列表 | 并列要点 | `- 第一点` |
| `1.` 列表 | 有序步骤 | `1. 第一步` |
| `---` 分隔线 | 大段落切换 | `---` |
| `💬`/`📌` emoji | 结尾 CTA 点缀 | 适量使用 |

### 段落长度控制

- 每段 **3-5 行**为宜，不超过 7 行
- 重要观点单句成段 = 视觉强调
- 段落间空行 — 移动端阅读需要呼吸感

## 两种创作路径

### 路径 A：脚本→文章（转化）

```
你说"把脚本003改成公众号文章"
  → 读取脚本 + 匹配知识卡片补充背景
  → 口播体→阅读体改写（扩展上下文、增加过渡）
  → 应用公众号排版（标题层级/引用/加粗/列表/分隔线）
  → 质检：humanizer-zh → risk-detector → ai-detector
  → 存 公众号文章/article-XXX-标题.md
  （与 短视频脚本/ 同级，存放在同一知识库下的 公众号文章/ 目录）
```

### 路径 B：独立创作（从零写）

```
你说"写一篇公众号文章，主题是XXX"
  → 读取痛点排行榜 + 关键词库 + 洞见卡 + 知识卡片
  → 按公众号结构创作（H1标题→导读→H2/H3分段→金句引用→CTA结尾）
  → 质检：humanizer-zh → risk-detector → ai-detector
  → 存 公众号文章/article-XXX-标题.md
  （与 短视频脚本/ 同级，存放在同一知识库下的 公众号文章/ 目录）
```
