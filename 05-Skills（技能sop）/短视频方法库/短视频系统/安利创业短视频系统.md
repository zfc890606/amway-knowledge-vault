       │  痛点排行榜         │

## 🎬 博主结构法（写脚本时的快速框架）

```
┌────────────────────────────────────────────────────────┐
│  ① 痛点开场（第一句就让观众对号入座）                   │

│  → 例："你是不是也觉得做安利就得天天刷朋友圈？"        │
├────────────────────────────────────────────────────────┤

│  1. ______                                              │
│  2. ______                                              │
│  3. ______                                              │
│  → 有具体数字、有场景、有对话，别讲道理                 │
├────────────────────────────────────────────────────────┤
│  ③ [可选] 讲原因（会掉播放量，但过滤精准用户）         │
│  "为什么你会有这个问题？因为…"                          │
│  → 想要播放量 → 删掉这步                                │
│  → 想要精准粉丝 → 保留这步                              │
├────────────────────────────────────────────────────────┤
│  ④ 一句话总结                                           │
│  "所以[一句话结论]"                                      │
│  → 不加"所以"，直接说                                   │
│  → 例："先让自己成为答案，别人自然来找你"               │
└────────────────────────────────────────────────────────┘

适用场景：所有"用户有具体问题→你有具体答案"的选题
使用建议：结构出来后在[[脚本风格系统]]做嘴替改写和风格自检
```
