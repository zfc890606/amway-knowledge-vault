## Day 1 — 知识库结构搭建

### 1.1 确定方法论

这套知识库基于博主「姜胡说」的个人知识管理方法论，核心理念是：

> **知识不是用来囤的，是用来「运行」的。**

五步闭环：**读 → 想 → 写 → 做 → 盘**

### 1.2 创建 PARA 目录结构

在 Obsidian 中新建库（或利用已有库），在根目录下建立 `知识库/` 文件夹，并创建以下子目录：

```
知识库/
├── 00-灵感库（标记灵感）/    ← 所有灵感的入口 ⭐
│   ├── index.md               ← 入盒规则（什么该收、什么不该收）
│   ├── 创作流程.md            ← 7刃筛选 → 挖矿 → 4K成卡
│   └── 知识卡片/              ← KC卡片存放
│       └── 卡片模板.md        ← 4K卡片模板
│
├── 01-Projects（进行的项目）/  ← 进行中的项目、待办项
│   └── 待办/
│
├── 02-Areas（关注的领域）/    ← 持续关注的领域
│
├── 03-Resources（参考素材资料）/ ← 参考资料 ⭐
│   ├── wiki/                  ← AI 知识编译（核心！）
│   │   ├── _index.md          ← Wiki 索引页
│   │   ├── 概念/              ← 概念解读卡片
│   │   └── 来源/              ← 原始来源摘录

│   ├── 短视频方法库/          ← 按分类存放方法技能
```

### 1.3 入盒规则（index.md）

在 [[知识库/00-灵感库（标记灵感）/index]] 中定义：

```markdown
## 入盒规则

✅ 收：
- 引发你思考的内容（不是让你认同的）
- 可转化为行动的方法论
- 打破你原有认知的观点
- 与当前关注主题相关的素材

❌ 不收：
- 纯资讯新闻（阅后即焚）
- 购物清单、明天吃啥
- "将来可能有用"但现用不上的东西
```

### 1.4 搭建要点

- **目录名带编号**（00-, 01-...）是为了在 Obsidian 文件列表中按自然顺序排列
- **命名带中文注释**（如 `00-灵感库（标记灵感）`）是为了中文用户一目了然
- `README.md` 是整个知识库的入口，写上快速链接索引

## Day 2 — Claude Code 环境初始化

### 2.1 安装 Claudian 插件

Claudian 是一个将 Claude Code（Anthropic 的 AI 编码助手）嵌入 Obsidian 的插件。

1. 打开 Obsidian → 设置 → 第三方插件 → 社区插件市场
2. 搜索 **Claudian**（插件 ID: `realclaudian`）
3. 安装并启用
4. 配置 API Key（需要 Anthropic API 或 Claude 订阅）

### 2.2 编写 CLAUDE.md

在 vault 根目录创建 `.claude/CLAUDE.md`，这是 Claude Code 的工作准则文件。它告诉 AI：

- **你是谁**：你的身份和角色
- **你有哪些知识库**：知识库位置、结构
- **你怎么工作**：行为准则

核心内容结构（完整文件参考现有 [[.claude/CLAUDE.md]]）：

```markdown
# Vault 工作准则

来源：[[知识库/05-Skills/短视频方法库/karpathy-guidelines]]

## 1. Think Before Coding — 先想再写
**不要假设。不要隐藏疑惑。摆出权衡。**

## 2. Simplicity First — 简单第一
**用最少的代码解决问题。**

## 3. Surgical Changes — 外科手术式改动
**只动必须动的。只清理自己造成的混乱。**

## 4. Goal-Driven Execution — 目标驱动执行
**定义成功标准。循环直到通过验证。**
```

### 2.3 配置 Obsidian 插件

```json
// .obsidian/community-plugins.json
[
  "realclaudian",           // Claude Code 集成
  "obsidian-excalidraw-plugin", // 手绘/图表
  "terminal",               // 终端集成
  "copilot"                 // AI Copilot
]
```

### 2.4 初始化的关键配置

| 配置项 | 位置 | 说明 |
|--------|------|------|
| `"sync": true` | `.obsidian/core-plugins.json` | 开启 Obsidian Sync |
| CLAUDE.md | `.claude/CLAUDE.md` | AI 工作准则 |
| 技能目录 | `.claude/skills/` | AI 技能存放位置 |
| 行为准则来源 | 知识库内 karpathy-guidelines | 可本地维护 |

## Day 3 — 技能库安装与集成

### 3.1 什么是"技能"（Skill）

Claude Code 的 **Skill** 是一份结构化的指令文件（SKILL.md），告诉 AI 如何完成特定类型的任务。技能存放在 `.claude/skills/` 目录下。

技能可以是：
- 一个独立的方法论（如 karpathy-guidelines）
- 一个工具的使用手册（如 markitdown）
- 一个完整的工作流定义（如 短视频系统）

### 3.2 安装方式

#### 方式一：npx 一键安装（推荐）

```bash
# 从 GitHub 安装技能到本地
npx skills add https://github.com/op7418/Humanizer-zh.git
```

这条命令会自动：
1. 将仓库克隆到 `~/.claude/skills/humanizer-zh/`
2. 技能立即可用

#### 方式二：Git Clone

```bash
git clone https://github.com/example/some-skill.git ~/.claude/skills/some-skill
```

#### 方式三：手动安装

1. 将技能文件夹复制到 `~/.claude/skills/` 下
2. 确保结构如下：
   ```
   ~/.claude/skills/my-skill/
   ├── SKILL.md       # 技能定义文件（必需）
   └── README.md      # 说明文档（推荐）
   ```

按分类列出（详细见 [[#附录 B — 技能清单]]）：

## Day 4 — 内容流水线建设

### 4.1 核心流水线

```
来源（微信/口述/直播/读书/思考）
    ↓  "这是灵感"
00-灵感库/  ← 入盒规则筛选
    ↓  7刃筛选
灵感炼金炉（挖矿8条）
    ↓  4K制式
知识卡片 KC → 知识卡片/
    ↓  积累充分
口播逐字稿 / 长文 / 短内容 → 06-输出/
```

### 4.2 七刃筛选（判断是否值得做）

在你把灵感做成知识卡片之前，先做 7 刃筛选。灵感来源（00-灵感库）中的每条记录都经过这 7 个问题过滤：

1. **真实吗？** — 来源可靠？信息有出处？
2. **有深度吗？** — 是表层信息还是底层逻辑？
3. **可迁移吗？** — 这个知识能用到其他地方吗？
4. **是我想的吗？** — 我真正关心这个话题吗？
5. **有行动吗？** — 知道这个能让我做什么不一样？
6. **有变现吗？** — 能帮到别人、产生价值吗？
7. **时机对吗？** — 现在做还是以后做？

记录在 `00-灵感库/创作流程.md` 中。

### 4.3 挖矿8条（从素材中提炼知识）

当一篇素材（文章、视频、音频）确定了要处理，用这 8 条原则挖出核心知识：

1. **概念** — 核心概念是什么？一句话解释
2. **因果链** — 什么导致什么？逻辑链条
3. **机制** — 为什么有效？背后的原理
4. **数据** — 关键数据/证据
5. **案例** — 具体事例
6. **反例** — 什么时候不适用
7. **关联** — 跟已有知识卡片的连接
8. **行动** — 我可以怎么用

### 4.4 4K 知识卡片

每张卡片遵循统一格式（模板见 [[知识库/00-灵感库（标记灵感）/知识卡片/卡片模板]]）：

```markdown
---
created: 2026-07-10
tags:
  - 知识卡片/[领域]
updated: 2026-07-10
---

# [概念名]

**一句话要点**
（一句话说清楚这是什么）

**背后的机制/逻辑**
（为什么是这样？底层原理是什么？）

**执行步骤/怎么用**
（具体怎么做？可操作的行动指南）

**边界条件/注意事项**
（什么时候不适用？常见错误？）
```

### 4.5 AI 辅助拆解流程

当有了素材后，可以这样让 AI 帮忙拆解成知识卡片：

```text
请帮我用知识卡片格式拆解这段内容：
[粘贴内容]

要求：
1. 提取核心概念，每个概念一张卡
2. 每张卡包含：一句话要点 → 底层逻辑 → 怎么用 → 边界条件
```

> **关键**：AI 负责"提取"，你负责"连接和应用"。

## Day 5 — 知识编译系统（Wiki）

### 5.1 什么是知识编译

知识编译是将**原始素材**转化为**结构化知识**的过程。类似维基百科——不保留原始文章，只保留经过提炼的概念和事实。

### 5.2 Wiki 结构

```
03-Resources（参考素材资料）/
└── wiki/
    ├── _index.md                 ← Wiki 总索引
    ├── 概念/                     ← 提炼后的概念解读
    │   ├── 慢性炎症与膳食平衡.md
    │   ├── 糖尿病逆转底层逻辑.md
    │   ├── 细胞更新周期.md
    │   ├── 轻断食与代谢修复.md
    │   ├── 细胞损伤四大链条.md
    │   └── 肝胆净化与护肝计划.md
    └── 来源/                     ← 原始来源摘要
        ├── 慢病细胞级根源与膳食赤字对冲报告.md
        ├── 身体自我修复时间线.md
        ├── 糖尿病逆转底层逻辑笔记.md
        └── 身心净化核心原理.md
```

### 5.3 编译流程

```
原始素材（笔记/文章/直播稿）
    ↓  提取核心概念
概念卡（多篇素材提炼一个概念）
    ↓  连接已有知识
更新 Wiki 索引（_index.md）
    ↓  积累
可交叉引用的知识网络
```

### 5.4 Wiki 索引页示例

[[知识库/03-Resources（参考素材资料）/wiki/_index]] 作为入口：

```markdown
# 🧠 AI 知识编译索引

> 编译好的概念知识，按主题检索。

## 慢性炎症
- [[慢性炎症与膳食平衡]]

## 代谢修复
- [[轻断食与代谢修复]]

## 细胞层面
- [[细胞更新周期]]
- [[细胞损伤四大链条]]
```

## Day 6 — 日常维护与迭代指南

> 知识库的核心不是「建完就完」，而是 **每天转动起来**。
> 下面这套迭代节奏参考了姜胡说的"读→想→写→做→盘"五步闭环，
> 把知识管理变成肌肉记忆，每天 15–30 分钟。

---

### 🔄 每日循环（15–30 分钟）

每天固定时间（建议早晨或睡前）跑一次：

```
⏰ 预计耗时：15–30 分钟
🛠 工具：Obsidian + Claudian
📋 主线：入盒 → 处理 → 输出 → 记录
```

#### Step 1 — 清空灵感箱（5 min）

打开 [[知识库/00-灵感库（标记灵感）/index]]，过一遍新入盒的内容：

```
□ 浏览所有新灵感（从微信/口述/读书/直播等渠道流入的）
    → 快速判断：这个值得做卡片吗？
    → 不值得 → 归档到 04-Archive/ 或直接删
    → 值得 → 进入 Step 2

例：
  灵感："今天直播有人说轻断食会不会掉肌肉"
  → 判断：值得做，这是常见疑问
  → 行动：记入 "待做卡片" 队列
```

#### Step 2 — 提炼一张卡片（10–15 min）

从灵感队列中选 1 条，走一次 **挖矿8条** → **4K格式**：

```
□ 选一条灵感
□ 用挖矿8条快速扫描（在心中过一遍）：
    → 概念是什么？
    → 因果链？
    → 机制？
    → 数据/证据？
□ 写入 4K 卡片模板
□ 存到 00-灵感库/知识卡片/
□ 加上 tag：#知识卡片/[领域]

⚠️ 注意：15 分钟写不完就停下来
  → 写个骨架 + TODO 标记
  → "完整版下次再补" 比 "等完美再发" 好 100 倍
```

#### Step 3 — 日志打卡（2 min）

打开 [[知识库/日志]]，做一条轻量记录：

```
□ 更新今天日期行：
   | 日期 | 输入灵感来源 | 处理成卡片 | 有没有输出 | 备注 |

例：
   | 07-11 | 直播问答×1 | 轻断食掉肌肉.md | 无 | 骨架已完成 |
```

### 📆 每月体检（45–90 分钟）

每月第一天或最后一天做一次深度体检：

```
⏰ 预计耗时：45–90 分钟
🛠 工具：Obsidian 内置功能 + 终端
📋 主线：健康检查 → 清理 → 归档 → 升级
```

#### 检查 1 — 知识库健康度扫描（15 min）

```
□ 断裂链接
    → Obsidian 左下角 → 「打开关系图谱」
    → 找"孤点"节点（没有任何连线的文件）
    → 判断：是真孤儿（可删除）还是缺链接（补上）

□ 孤儿笔记
    → 设置 → 核心插件 → 启用「未链接笔记」
    → 扫描结果列表
    → 每条判断：链接到相关卡片？归档？删除？

□ 空文件/残留
    → 终端：find ./知识库 -type f -name "*.md" -empty
    → 清理不再需要的空文件
```

#### 检查 2 — 卡片库质量审查（20 min）

```
□ 重复/重叠卡片
    → 扫描知识卡片目录，找主题相似的文件
    → 判断：合并成一张？保留各自角度？删掉弱的？

□ 过时内容
    → 检查 created/updated 日期超过 30 天的卡片
    → 内容还准确吗？有新的认知需要补充吗？
    → 补充后更新 updated 字段

□ 无标签卡片
    → 终端：grep -rL "^tags:" 知识库/00-灵感库/知识卡片/*.md
    → 给没有 tags 的卡片补上分类标签
```

#### 检查 3 — 目录结构审核（10 min）

```
□ 00-灵感库 里的灵感有没有堆积超过 2 周的？
    → 处理或归档，保持灵感箱不超过 10 条未处理

□ 01-Projects 里有没有已完成但没归档的项目？
    → 移动到 04-Archive/

□ 06-输出/ 里的成品是否散乱？
    → 按月建子文件夹（2026-07/）整理输出文件
```

#### 检查 4 — 技能库健康度（10 min）

```
□ 检查技能目录完整性：
    ls ~/.claude/skills/  — 每个目录下都有 SKILL.md 吗

□ 检查 GitHub 上游更新（对每个带 .git 的技能目录）：
    cd ~/.claude/skills/<name> && git remote -v && git log --oneline -3

□ 检查 CLAUDE.md 中的技能引用是否过时：
    有技能已安装但没写在 CLAUDE.md 里？→ 补上
    有技能写在 CLAUDE.md 里但已删除？→ 移除
```

#### 检查 5 — 备份验证（5 min）

```
□ Obsidian Sync 状态：设置 → 同步 → 验证最近同步时间
□ 本地有无近期 .obsidian/backup/ 自动备份
□ 知识库总大小：du -sh 知识库/ — 是否异常增长
```

### 🧠 迭代节奏总览

| 频率 | 耗时 | 核心动作 | 产出 |
|------|------|----------|------|
| **每日** | 15–30 min | 入盒 → 1 张卡片 → 日志 | 每天 1 张卡片，知识不积压 |
| **每周** | 30–60 min | 串卡成文 → 更新 Wiki → 复盘 | 每周 1 篇输出，知识变作品 |
| **每月** | 45–90 min | 健康检查 → 清理 → 归档 | 系统保持整洁，不积重难返 |

> **关键心态**：不要追求每天完美执行。
> 每天写了卡片就是胜利，每周有输出就是丰收，每月做了体检就是长期主义。
> 断了一天不要紧，断了一周捡起来继续——系统不惩罚重启。

```bash
ls ~/.claude/skills/

# 更新单个技能
cd ~/.claude/skills/skill-name && git pull

# 批量更新所有技能（带 git 仓库的）
for d in ~/.claude/skills/*/; do
  [ -d "$d/.git" ] && (cd "$d" && echo "→ Updating $(basename $d)..." && git pull)
done

# 安装新技能
npx skills add https://github.com/username/skill-repo.git

# 卸载技能（直接删除目录）
rm -rf ~/.claude/skills/unwanted-skill
```

## 附录 C — 故障排除

### 1. 技能装不上

```bash
# 检查 Git 是否可用
git --version

# 手动克隆
git clone <repo-url> ~/.claude/skills/<name>

# 确认 SKILL.md 存在
ls ~/.claude/skills/<name>/SKILL.md
```

### 2. AI 不按知识库结构工作

- 检查 `.claude/CLAUDE.md` 是否正确引用了知识库路径
- 确认 wikilink 写法正确（如 `[[知识库/README]]`）
- 尝试在对话中直接说："请先读取 CLAUDE.md 了解知识库结构"

### 3. Obsidian Sync 不同步

- 检查设置 → 同步 → 连接状态
- 确保库路径是 Obsidian 识别的库（而非独立文件夹）
- 重启 Obsidian 尝试

### 4. 技能加载慢

- 技能目录过大（`banana-slides` 带 `.venv` 这种重文件）会影响加载
- 考虑将大技能从 `.claude/skills/` 移到 `知识库/05-Skills/` 作为参考
- 在 CLAUDE.md 中用 wikilink 引用知识库中的方法
