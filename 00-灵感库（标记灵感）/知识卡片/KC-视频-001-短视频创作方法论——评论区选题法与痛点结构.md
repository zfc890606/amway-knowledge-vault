# 《短视频创作方法论——评论区选题法与痛点结构》

> **一句话本质**：
> 选题不是靠"抄爆款"，是去评论区统计真问题。内容结构不是"钩子套钩子"，是痛点→方案→原因（过滤）→总结。视频不是"烟花"，是越久越值钱的搜索资产。

---

## 🧩 因果 / 原理

### 它解决了什么问题？

**选题枯竭：抄爆款抄到没得抄**

大多数创作者的选题流程：
```
网上找爆款 → 让AI分解 → 抄过来 → 发出去 → 没人看 → 再找下一个爆款
```
这个循环的问题：
- 很快就没选题了（爆款就那么多）
- 内容跟你没有关系（不是你自己的东西）
- 用户一眼就能看出"这是抄的"

**播放量陷阱：追求推荐流量 → 忽视搜索资产**

推荐流量像烟花：biu~ 半个月就没了。
搜索流量像资产：一直有、一直有、持续积累。

### 为什么有效？

**评论区=需求雷达**

真需求长什么样？
- 用户愿意花时间打出来的问题 → 是他真实卡住的点
- 重复出现的问题 = 高频痛点
- **付费问题 = 最好的选题**（愿意花钱问 = 真的想要答案）

所以选题的底层逻辑是：
```
不去猜用户想要什么 → 去听用户正在问什么
```

**痛点→方案→原因→总结 结构的原理**

| 环节 | 作用 | 为什么有效 |
|------|------|-----------|
| 痛点开场 | 让用户对号入座 | "你是不是也有这个问题？" → 说的就是我 |
| 给方案 | 满足需求 | 12345步，清晰可执行 |
| 讲原因 | 过滤精准用户 | 愿意听原因的=真想做的人=你的目标用户 |
| 一句话总结 | 记忆锚点 | 让用户记住你 |

**至于为什么要加"原因"（即使它会降播放量）**：
因为你在做**IP**，不是在做**流量**。
- 不要精准用户 → 去掉原因，播放量更高
- 要精准用户 → 保留原因，过滤掉"白嫖党"

## 🛠 可执行方法

### 最小行动（今天就能做）

打开任何一个对标账号的评论区，收集前10个问题，统计频率 → 你就有了选题。

### 完整执行步骤

```
┌─ 一、选题方法 ──────────────────────────────────────────┐
│                                                           │
│  ① 建需求池：                                            │
│     ├─ 自己的社群/私信 → 记录每一条问题                   │
│     ├─ 对标账号评论区 → 爬取高频问题                      │
│     └─ 付费平台（知乎付费咨询/知识星球等）→ 付费问题=好选题 │
│                                                           │
│  ② 统计频次：                                            │
│     └─ 同样的问题出现几次？→ 按频次排榜                    │
│                                                           │
│  ③ 选题从排行榜取：                                       │
│     └─ 排名越靠前=越多人痛=越优先拍                        │
│                                                           │
│  ④ 关键词策略：                                          │
│     └─ 用户怎么问，你就怎么答（不要用高大上词）            │
│        人们说"卡在500播放量" → 你标题就写"播放量卡500"     │
│        不说"算法推荐瓶颈"                                 │
│                                                           │
└──────────────────────────────────────────────────────────┘

┌─ 二、内容结构 ──────────────────────────────────────────┐
│                                                           │
│  ① 痛点开场                                              │
│     "你是不是也[具体问题]？"                               │
│     → 第一句就让观众对号入座                               │
│                                                           │
│  ② 给解决方案（12345）                                    │
│     "我告诉你几个方法，我是这么做的"                       │
│     → 清晰可执行，不要讲大道理                             │
│                                                           │
│  ③ [可选] 讲原因                                          │
│     "为什么你会有这个问题？"                               │
│     → 会掉播放量，但会过滤出精准用户                       │
│     → 做IP就保留，做流量就删掉                             │
│                                                           │
│  ④ 一句话总结                                             │
│     → 让用户记住你                                          │
│                                                           │
└──────────────────────────────────────────────────────────┘
```

### 搜索资产观（区别于推荐流量观）

| 推荐流量 | 搜索流量 |
|---------|---------|
| 平台推→用户刷到 | 用户搜→主动找到 |
| 半个月就没了 | 一直在，持续增长 |
| 看缘分 | 看你的关键词布局 |
| 像烟花 | 像资产 |

**怎么做搜索资产**：
- 用日常语言做标题/描述（人们怎么说就怎么写）
- 不追求标题党，追求"搜得到"
- 一条视频发布半年后才是它的真正价值开始

## 📍 场景与边界

### 适用场景

- **任何赛道的短视频创作** — 评论区需求池+痛点结构，通用
- **想做个人IP不是纯流量号** — 保留"原因"环节过滤用户
- **冷启动阶段** — 没有爆款经验时，先做搜索资产积累

### 边界条件

- **没有评论区可扒**：新号/新赛道 → 去知乎/小红书/公众号评论区代替
- **纯流量号不要加原因**：如果目标是最大化播放量，删掉"讲原因"部分
- **不能只依赖搜索**：搜索+推荐两条腿走路，比例7:3还是3:7取决于阶段

### 常见失败原因

1. 只抄爆款不挖评论区 → 选题枯竭
2. 痛点不够痛 → "怎么营销"太大了，"播放量卡500"才是真痛点
3. 结构没有痛点→方案→总结 → 观众看完不知道要干嘛
