## ════════ 流水线A：知识加工 ════════

### 📥 输入 → 入盒 → 成卡 → 反哺

```
你随时丢内容过来（链接/文章/思考/问题/录音）
       │
       ▼
┌─ ① 入盒记录 ──────────────────────────────────┐
│  写入 00-灵感库/index.md 「最新灵感」+ 标注日期   │
└────────────────────┬──────────────────────────┘
                     ▼
┌─ ② 查重 ──────────────────────────────────────┐
│  ⚠️ 先看当下有没有相同或类似的？                   │
│  ├─ 有 → 跳过（不进卡片/洞见卡/选题）            │
│  └─ 没有 / 有必要加 → 继续往下                   │
└────────────────────┬──────────────────────────┘
                     ▼
┌─ ③ 判断分流 ───────────────────────────────────┐
│  ├─ 有深度认知 → 知识卡片（KC-板块-编号）         │
│  ├─ 有金句洞见 → 洞见卡（DJ-板块-编号）           │
│  ├─ 有选题价值 → 更新 选题库                     │
│  └─ 归入板块   → 更新对应 index.md              │
└────────────────────┬──────────────────────────┘
                     ▼
┌─ ④ 并行反哺 ───────────────────────────────────┐
│  路① 知识沉淀 → 归类4大板块 + 更新索引            │
│  路② 选题金句 → 选题库 + 洞见卡 → 触发流水线B    │
│  路③ 项目反哺 → 更新待办文件 → 流入流水线C       │
└────────────────────────────────────────────────┘
```

### 📂 三大原材料库 + 两个输出库（分清输入/输出）

```
流水线A自动产出（输入侧）：
│
├─ ① 查重 ────────────────────────────────────────────
│  ⚠️ 加之前先看：当下有没有相同或类似的？
│  ├─ 有 → 跳过，不再重复建
│  └─ 没有 / 有必要加 → 继续往下
│
├─ ② 产出 ────────────────────────────────────────────
│  ├── 知识卡片/   ← 📇 KC-板块-编号-主题 格式
│  │                 完整的认知框架（因果/原理/方法/场景）
│  ├── 洞见卡/     ← 💎 DJ-板块-编号-DJ-主题 格式
│  │                 一句话洞见+可用方法（写脚本时取金句）
│  │                 📌 已整合原金句素材库，现共 86 张
│  └── 选题库.md   ← 🔥 按痛点频率排榜的选题方向
│                    写脚本前来这里看拍什么
│

流水线B产出（输出侧，需你指令）：
├── 脚本库/         ← 🎬 你说"写脚本"后存入
└── 公众号草稿/     ← 📱 你说"写公众号/痛点#XX写成公众号/脚本改公众号"后 wewrite 全自动产出
```

> **自动执行规则**：任何内容进来，不需问用户，直接跑完整流程。
> 飞书录音：用户说"拉飞书的录音"才触发，不后台自动跑。
